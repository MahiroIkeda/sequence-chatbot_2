# SpecFlow — 仕様駆動開発アシスタント

信州大学 池田眞浩（2025）が研究・開発したツール。  
要件定義書からシーケンス図を自動生成し、GitHubと連携して仕様とコードを同期管理する。

> 仕様駆動開発の理論的背景・原則・7工程との対応については → [docs/SPEC_DRIVEN.md](docs/SPEC_DRIVEN.md)

---

## Why（なぜ作ったか）

自然言語で書かれたドキュメントは時間とともに陳腐化する。コードが変わっても仕様書が追従しないまま放置される状況は、ソフトウェア開発における根本的な問題の一つである。

本ツールは「仕様駆動開発（Specification-Driven Development）」の原則に基づき、仕様とコードを常に同期させることを目的として設計した。要件定義書をアップロードするだけでAIが曖昧性を検出し、シーケンス図を自動生成してGitHubに保存するまでの一連のフローを一つのUIで完結させている。

---

## What（何をするか）

- `.txt` 形式の要件定義書をアップロードする
- AIが曖昧性を自動検出し、形式的定義で補完する
- 仕様を明確性・完全性・一貫性・実装可能性の4観点でレビューする
- Mermaid記法のシーケンス図を自動生成・表示・編集する
- チャットで修正依頼を出し、図をインタラクティブに改善できる
- GitHubのデフォルトブランチに仕様書・シーケンス図・READMEを自動コミットする
- Pull Requestを自動生成し、仕様変更の差分をAIが解説する
- 前回保存した仕様との差分を検出し、影響範囲を分析する

## セットアップ

### 1. 必要なライブラリのインストール

```bash
pip install flask anthropic python-dotenv
```

### 2. APIキーの取得

**Anthropic APIキー**
1. https://console.anthropic.com にアクセスしてアカウントを作成
2. 左メニューの「API Keys」→「Create Key」でキーを発行
3. `sk-ant-...` の形式の文字列をコピーしておく

**GitHub Token**
1. GitHub にログインし https://github.com/settings/tokens にアクセス
2. 「Generate new token (classic)」をクリック
3. `repo` にチェックを入れて「Generate token」
4. `ghp_...` の形式の文字列をコピーしておく（画面を閉じると二度と表示されないので注意）

### 3. .envファイルの作成

プロジェクトルート（`app.py` と同じフォルダ）に `.env` という名前のファイルを作成し、以下を記述する。

```
ANTHROPIC_API_KEY=sk-ant-ここにAPIキーを貼り付ける
GITHUB_TOKEN=ghp_ここにGitHubトークンを貼り付ける
GITHUB_OWNER=自分のGitHubユーザー名
GITHUB_REPO=保存先のリポジトリ名
```

> **注意**: `.env` はGitの管理対象から除外されており、GitHubには公開されない。APIキーを他人に見せたり、コードに直接書いたりしないこと。

### 4. サーバーの起動

```bash
python app.py
```

### 5. ブラウザでアクセス

```
http://localhost:5000
```

## 使い方

1. **要件定義書をアップロード**（.txt形式）
2. **「曖昧性を自動検出」** — AIが曖昧な箇所を特定し、形式的定義で補完する
3. **「仕様をレビュー」** — 4観点でレビューし、改善済み仕様を生成する
4. **「シーケンス図を生成」** — Mermaid記法の図を自動生成・表示する
5. チャット欄で修正依頼を出し、図を調整する
6. **「差分を検出」** — 前回保存との変更点をAIが解説する
7. **「GitHubに保存」** または **「Pull Request」** で仕様をリポジトリに反映する

## 研究背景

本ツールの設計には、学部卒業研究「対象ドメイン定義に数学構造を含めてLLMに注入することによる推論性能の向上と検証」（2025）の知見を活用している。自然言語だけで複雑なルールをLLMに伝えると論理破綻が生じやすいという問題に対し、ドメインを形式的に定義してから注入することで推論精度が向上することを実験的に示した。この知見をもとに、SpecFlowでは曖昧性を形式的補完してからシーケンス図を生成する設計としている。

大学院では引き続き、仕様駆動開発とハーネスエンジニアリングを組み合わせたAIエージェントの信頼性向上を研究テーマとしている。

- GitHub Secretsへのシークレット管理移行
