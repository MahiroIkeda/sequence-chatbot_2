Pull Request フローについて
仕様駆動開発の原則3「仕様は変更と反復が前提」を実践するフローです。 1. SpecFlowで仕様を更新・シーケンス図を生成 2. 「Pull Requestを作成する」→ 新しいブランチが自動作成される 3. AIがPR説明文（変更理由・影響範囲・レビューポイント）を自動生成 4. GitHubでチームがレビュー・コメント・承認 5. 承認後にマージ → 変更履歴として永続的に記録 この承認フローにより「誰が・いつ・なぜ仕様を変更したか」が追跡可能になります。
GitHubに保存
