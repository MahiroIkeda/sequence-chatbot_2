# 要件定義書：図書館蔵書管理システム v1.1

## 1. Why（なぜ作るか）

### 1.1 背景
現在、○○市立図書館（蔵書数約10,000冊、会員数約500名）では、図書の貸出・返却管理を紙台帳で行っており、以下の課題が発生している。

**現状の課題**
- 蔵書の在庫状況確認に平均5〜10分かかり、利用者を待たせている
- 紙台帳のため貸出履歴の検索が困難
- 延滞管理が手作業で漏れが発生
- 人気書籍の予約管理が煩雑
- 統計データの集計に月末1日を要する

### 1.2 目的
システム化により以下を実現する。

**定量的目標**
- 蔵書検索時間を30秒以内に短縮
- 貸出・返却処理を1件あたり1分以内に短縮
- 延滞管理の自動化により司書の事務作業を週5時間削減
- 貸出率を現状の60%から75%に向上

**定性的目標**
- 利用者の利便性向上（いつでも在庫確認・予約可能）
- 司書の業務効率化（データ入力・集計作業の削減）
- 蔵書の有効活用促進（予約機能による取り置き、統計分析）

### 1.3 対象範囲
- **Phase 1（本要件定義の対象）**：基本的な貸出・返却・予約機能
- **Phase 2（将来対応）**：電子書籍、モバイルアプリ、他図書館との連携

## 2. 用語定義

| 用語 | 定義 |
|------|------|
| 利用者 | 図書館サービスを利用する一般市民。会員登録後は「会員」と同義 |
| 会員 | 利用者登録を完了し、会員IDを付与された利用者 |
| 司書 | 図書館職員。システムの管理者権限を持つ |
| 蔵書 | 図書館が所有する書籍。同一タイトルで複数冊存在する場合は個別の蔵書IDで管理 |
| 書誌情報 | 書籍の基本情報（タイトル、著者、ISBN、出版社など）。同一書籍の複数冊で共有 |
| 在庫状態 | 個々の蔵書の物理的な状態。「貸出可能」「貸出中」「取り置き中」「廃棄済み」の4状態 |
| 予約状態 | 蔵書に対する予約の有無。「予約なし」「予約あり（待機中）」「予約あり（通知済み）」の3状態 |
| 延滞 | 貸出期限を過ぎても返却されていない状態 |
| 取り置き | 予約者のために一時的に貸出を制限している状態（通知後3日間） |
