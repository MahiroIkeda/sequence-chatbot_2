# 要件定義書：図書館蔵書管理システム（改訂版）

## Why（なぜ作るか）

現在、図書館の貸出・返却管理は紙台帳で行っており、蔵書の状況把握に時間がかかっている。
システム化することで、利用者と司書の双方の負担を減らし、蔵書の有効活用を促進する。

---

## What（何をするか）

### 1. 利用者登録

#### 1.1 登録処理
利用者は以下の情報を入力して会員登録を行う：
- 氏名（必須、最大100文字）
- メールアドレス（必須、RFC 5322準拠）
- 電話番号（必須、ハイフンなし10〜11桁の数字）

#### 1.2 利用者ID発行
登録完了時、システムが一意の利用者ID（正の整数）を自動発行する。

#### 1.3 重複登録の制約
**形式的定義**:
```
∀u1, u2 ∈ User: 
  (u1.email = u2.email ∧ u1.status = ACTIVE ∧ u2.status = ACTIVE) ⟹ u1 = u2
```
- 同一メールアドレスで有効（status = ACTIVE）な利用者は1名のみ存在できる
- 退会済み（status = INACTIVE）の利用者と同じメールアドレスでの新規登録は可能
- 新規登録時は新しい利用者IDが発行される

---

### 2. 蔵書検索

#### 2.1 検索条件
利用者は以下の条件で蔵書を検索できる：
- タイトル（部分一致）
- 著者名（部分一致）
- ISBN（完全一致、ハイフンなし13桁）
